# Post-Codegen Fixes Documentation

This document describes the post-generation script created to fix systemic TypeScript errors in Telescope-generated code.

## Overview

The [`scripts/post-codegen-fix.ts`](../scripts/post-codegen-fix.ts) script automatically fixes known issues in code generated by Telescope (which uses `@cosmology/ast`). It runs automatically after code generation via the `generate` npm script.

## Fixed Issues

### 1. Scalar Type Mapping ✅ FIXED

**Problem:** The generator outputs invalid `double` type references instead of TypeScript's `number` type.

**Examples:**
- `[key: string]: double;` → `[key: string]: number;`
- `double.fromPartial()` → `Number()`
- `double.toAmino(v)` → `v`

**Impact:** Completely fixed across all 425 generated TypeScript files.

### 2. RPC Client Type Mismatch ✅ FIXED

**Problem:** Generated RPC query files import and use the deprecated `Tendermint34Client` instead of `CometClient`.

**Fix:**
```typescript
// Before
import { Tendermint34Client, HttpEndpoint } from "@cosmjs/tendermint-rpc";
const tmClient = await Tendermint34Client.connect(rpcEndpoint);

// After
import { CometClient, HttpEndpoint } from "@cosmjs/tendermint-rpc";
const tmClient = await CometClient.connect(rpcEndpoint);
```

**Impact:** Fixed in 2 files:
- `src/codegen/cosmos/rpc.query.ts`
- `src/codegen/lumera/rpc.query.ts`

### 3. Nested Message Defaults ⚠️ GENERATOR BUG

**Problem:** The `@cosmology/ast` generator incorrectly marks optional message fields as required in TypeScript interfaces.

**Example:**
```typescript
// Interface marks field as required (incorrect)
export interface GenesisState {
  params: Params;  // Should be: params?: Params;
}

// But fromPartial correctly treats it as optional
fromPartial(object: I): GenesisState {
  const message = createBaseGenesisState();
  message.params = object.params !== undefined && object.params !== null 
    ? Params.fromPartial(object.params) 
    : undefined;  // TS Error: Type 'undefined' is not assignable to type 'Params'
  return message;
}
```

**Why Not Fixed:** This is a fundamental bug in the AST generator's interface generation logic. Attempting to "fix" it by changing `undefined` to `createBaseType()` would:
1. Create runtime overhead (unnecessary object creation)
2. Potentially cause logic bugs (non-null checks would pass when they shouldn't)
3. Mask the real issue (incorrect interface definitions)

**Current Status:** ~200 TypeScript errors remain related to this issue. These are compilation warnings that don't affect runtime behavior since the actual values ARE correctly optional.

## Usage

The fix script runs automatically as part of code generation:

```bash
# Runs both codegen and fixes
pnpm generate

# Or run separately
pnpm run codegen        # Generate code
pnpm run codegen:fix    # Apply fixes
```

## Remaining Issues

**TypeScript Compilation Errors:** ~200 errors remain due to the nested message defaults issue described above.

**Recommended Solutions:**
1. **Short term:** Use `// @ts-expect-error` comments or TypeScript's `skipLibCheck` option
2. **Medium term:** Report the bug to the Telescope/cosmology team with examples
3. **Long term:** Wait for an upstream fix in `@cosmology/ast` or patch the generator directly

## Technical Implementation

The script uses:
- Node.js `fs` module for file I/O
- `glob` for file discovery
- Regex patterns for targeted replacements
- Conservative approach to avoid breaking changes

### Fix Functions

1. **`fixScalarTypes()`** - Replaces all `double` type references with `number`
2. **`fixRpcClientType()`** - Updates `Tendermint34Client` to `CometClient` 
3. **`fixNestedMessageDefaults()`** - Currently disabled due to generator bug

## Maintenance

When updating Telescope or `@cosmology/ast`:
1. Run `pnpm generate` to regenerate code
2. Check script output for any new patterns
3. Update fix patterns if needed
4. Test build with `pnpm build`
