    Interface IndexFile

    Index file structure for LEP-1 Cascade uploads.

    The index file contains metadata about the RaptorQ layout and is signed by the uploader to ensure data integrity. It includes 50 derived layout IDs and a cryptographic signature over the layout data.

    Remarks

    This structure must be serialized deterministically (canonical JSON) when computing signatures to ensure cross-platform compatibility.

    layout_ids

    layout_ids: string[]

    Array of 50 derived layout IDs.

    Remarks

    Derived using the LEP-1 algorithm: Base58(BLAKE3(zstd(layout_with_signature + "." + counter))) where layout_with_signature = Base64(layout_file) + "." + Base64(layout_signature) and counter = rq_ids_ic + i for i in [0, rq_ids_max)

    layout_signature

    layout_signature: string

    Cryptographic signature over the layout data.

    Remarks

    Generated by signing the canonical JSON bytes of the layout using ADR-036 signArbitrary with the uploader's wallet. Stored as Base64-encoded bytes.
