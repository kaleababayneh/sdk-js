# RaptorQ WASM Browser Bindings

This directory contains WebAssembly (WASM) files for using the RaptorQ library in web browsers.

## Files

- `rq_library.js` - The JavaScript bindings generated by wasm-bindgen
- `rq_library.d.ts` - The TypeScript definitions generated by wasm-bindgen
- `rq_library_bg.wasm` - The compiled WebAssembly binary
- `test-mem-module.html` - An example HTML file demonstrating how to use the library

## Usage

Import the `RaptorQSession` class directly from the generated JavaScript bindings:

```javascript
import { RaptorQSession } from './path/to/dist/lib/wasm/browser/rq_library.js';
```

Create a new session and use the API:

```javascript
// Create a new session
const session = new RaptorQSession(
  65535,          // symbol_size
  10,             // redundancy_factor
  1024,           // max_memory_mb
  4               // concurrency_limit
);

// Initialize a filesystem implementation
await session.set_filesystem(fsImplementation);

// Create metadata
const metadata = await session.create_metadata(
  '/path/to/input.file', 
  '/output/dir', 
  0,              // block_size (0 = auto)
  true            // return_layout
);

// Encode a file
const result = await session.encode_file(
  '/path/to/input.file',
  '/output/dir',
  0               // block_size (0 = auto)
);

// Decode symbols
await session.decode_symbols(
  '/symbols/dir',
  '/path/to/output.file',
  '/path/to/layout.json'
);

// Get recommended block size
const blockSize = await session.get_recommended_block_size(fileSize);

// Get library version
const version = await RaptorQSession.version();

// Clean up when done
await session.destroy();
```

See the `demo-wrapper.html` file in this directory for a complete example of how to use the library in a browser environment.

## Virtual Filesystem

The RaptorQ library requires a filesystem implementation for reading and writing files. In a browser environment, you need to provide a virtual filesystem implementation that implements the following methods:

- `readFile(path)`: Read a file and return a Promise that resolves to a Uint8Array
- `writeFile(path, data)`: Write a Uint8Array to a file and return a Promise
- `mkdir(path)`: Create a directory and return a Promise
- `stat(path)`: Get file or directory information and return a Promise that resolves to an object with `size` and `isDirectory()` properties

See the example implementation in `examples/wasm-browser-demo/demo-wrapper.html`.
